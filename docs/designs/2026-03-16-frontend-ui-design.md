# 技能驱动的 Agent 平台前端 UI 设计 (v1 风格候选)

- 日期：2026-03-16
- 状态：已定稿 (Finalized v1)

## 核心设计底线 (不可违背原则)
1. **纯正蓝白色调**：背景为白/浅灰，主交互色为“科技蓝/商务蓝”（如 `#1677FF`），禁止使用紫色、紫红、渐变及荧光色。
2. **拒绝大圆角**：全面禁止 `12px` 及以上的半圆角/胶囊按钮。全局控件（按钮、卡片、输入框）仅使用 `0px`（直角）至最大限度的 `4px`（微倒角）的边框半径。
3. **去 "AI 味"**：禁止闪烁发光效果（Glow）、无需意义的粒子背景、打字机动效和“魔法棒”类拟人化 Icon，保持纯粹的数据工具或 IDE 质感。

## 定稿界面布局 (Style F: Hybrid A+D)

经过充分考量和迭代，最终选择了结合“经典数据库工具风 (Classic IDE)”与“分析仪分割面板风 (Split-Pane Dashboard)”优势的混合布局方案 (Style F)。此方案兼具了工程严谨性和直观的数据检查能力。

### 1. 全局布局特点 (3-Column Layout)
采用极致紧凑的三栏硬边界布局：
- **左侧 (Explorer)**：`256px` 或 `w-64` 固定宽度的会话夹和 Skill 选择器，极简扁平。
- **中间 (Main Stage)**：占据主体的核心对话文本流与 SQL/指令编辑器。
- **右侧 (Sandbox Inspector)**：占据 `35%` 宽幅（最小 `350px`）的专属常驻数据浏览沙盒。

### 2. 视觉表现层
- **纯粹配色**：全局采用纯白 (`#FFFFFF`) / 浅灰 (`#FAFAFA`) 作为背景体系。用户态激活采用严谨的淡蓝底色 (`#EFF6FF`) 和淡蓝描边 (`#BFDBFE`)。
- **0圆角主义**：整个界面的容器、面板、输入框均严格奉行 `0px` 或 `2px` 圆角，消除玩具感。
  - **特例**：仅针对中间对话流中的“User Message 文本块”开放微倒角 ( `8px`) 借此达成最佳的可读边界区分。
- **极细分割线**：各区域严格依靠 `solid 1px #E0E0E0` 或 `#D9D9D9` 进行硬物理级切割，禁止软连接、无意义的投影 (Box-Shadow)。

### 3. 会话流核心交互 (Message Stream & Thinking)
- **Agent 思考过程收纳**：禁止平铺展示 Agent 大段的工具调用或 JSON 数据。所有推理 (`reasoning`) 和方法调用 (`tool_call`) 全部归约至一个带有 `▶` 旋转折叠/展开图表的 `<details>` 区块（"Thinking Process"）内。仅直接渲染 Agent 的终局文本结果 (`text`)。
- **数据关联穿透**：左侧/中间完全不绘制定制化表格或大图表。每个 Tool Call 数据流结果，只需展示一句摘要和一个明显的链接（`Inspect in Sandbox →`）。点击即刻将其完整 JSON/Table 数据传递至**右侧面板**呈现。
- **左右站位隔离**：模仿经典的 Chat 直觉：Agent 数据占据主区块且居左；User 发言强制靠右并在淡蓝色框内渲染，进一步切断视觉粘连。

这一套方案完美隔离了“沟通指挥（左+中）”与“深度数据考察（右）”，将该系统严格定义为具备专家级核查能力的平台级工具。
